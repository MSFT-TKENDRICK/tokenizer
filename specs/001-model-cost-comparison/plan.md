# Implementation Plan: Model Cost Comparison Panel

> Generated by `/speckit.plan`. Records the technical approach the squad will
> follow. Decisions that revise the spec are captured as ADRs under
> `docs/sdlc/adr/`; focused enrichments are captured as micro-spec facets under
> `docs/sdlc/facets/`.

## Technical Context

- **Stack:** React + Vite + TypeScript PWA (see repo root).
- **Reused modules:**
  - `src/lib/tokenizer.ts` — `tokenize`, `summarizeTokens`, `estimateContextPercentage`.
  - `src/lib/copilotModels.ts` — `COPILOT_MODEL_OPTIONS`, `estimateInputUsd`, `usdToAiCredits`.
- **Design system:** all UI work follows `DESIGN.md` tokens/components; run `npm run design:lint` after edits.

## Approach

1. Add a pure `compareModels(text, models)` helper next to `copilotModels.ts` that
   returns, per model, `{ usd, credits, contextPercentage, fits }`.
2. Render a `ModelComparisonPanel` React component: a sortable, semantic `<table>`
   driven by that helper.
3. Persist the selected model subset via `localStorage` behind a small typed wrapper.
4. Memoise the comparison so re-renders stay within the NFR-002 budget.

## Constraints

- **CON-001**: All currency math MUST reuse `copilotModels.ts` helpers; no duplicated pricing constants. [cap:cost-estimation]
- **CON-002**: No new runtime dependencies may be added for this feature. [cap:comparison]

## Testing Strategy

- Unit (vitest): `compareModels` math, sorting, fits/over-budget flags.
- BDD: one feature file per capability (generated under `docs/sdlc/`).
- E2E (Playwright): table renders, sorts, highlights over-budget models, persists selection.
- A11y (axe): table semantics + contrast.

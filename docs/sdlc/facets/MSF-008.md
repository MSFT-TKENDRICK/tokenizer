# MSF-008 — Acceptance criteria for FR-006: model comparison table + sorting

> Micro-spec facet: a focused enrichment of the SpecKit spec, generated for an
> identified ambiguity to give squad members the knowledge they need to execute.

**Grounded in:** FR-006 (consumes FR-005 cost math; feeds FR-007 over-limit highlight)

## Question (from interview)

AMB-003 (HIGH, missing-acceptance-criteria): what are the explicit pass/fail criteria
for FR-006 ("a comparison table with one row per model … sortable by input cost and by
context usage")?

## Resolution

The comparison table is driven by `COPILOT_MODEL_OPTIONS` (from the ported
`web/models.mjs`, a 1:1 port of `src/lib/copilotModels.ts`). There are **23 rows**:
the synthetic `Auto` model plus the 22 entries of `COPILOT_MODELS` (verified against the
current catalog). Each row shows: model **name**, **provider**, **input cost in USD**
(`estimateInputUsd(tokenCount, model)`), **input cost in AI credits**
(`estimateInputAiCredits` = `usdToAiCredits(usd)`, i.e. USD / 0.01), and
**context-window usage** (`tokenCount / model.contextWindow`, as a percentage capped at
100). Sorting is user-controllable by **input cost** and by **context usage**, ascending
and descending, and the sort is stable.

Markup must be a semantic `<table>` with a header row (cross-references NFR-001 a11y).

## Acceptance criteria

- [ ] The table renders exactly one row per entry of `COPILOT_MODEL_OPTIONS` (23 rows:
      `Auto` + 22 catalog models) with columns: name, provider, input USD, input credits,
      context usage.
- [ ] For each row, USD = `estimateInputUsd(tokenCount, model)` and credits =
      `estimateInputAiCredits(tokenCount, model)` = USD / 0.01 (within floating-point
      tolerance) for the current token count.
- [ ] Clicking the "input cost" header sorts rows by USD ascending; clicking again sorts
      descending; the displayed order matches a numeric sort of the cost values.
- [ ] Clicking the "context usage" header sorts rows by `tokenCount/contextWindow`
      ascending then descending; ties preserve catalog order (stable sort).
- [ ] The table uses semantic `<table>`/`<thead>`/`<th scope>` markup with a header row
      (verified jointly with NFR-001).

# MSF-006 — Resolve FR-009: session-scoped persistence of the canvas input

> Micro-spec facet: a focused enrichment of the SpecKit spec, generated for an
> identified ambiguity to give squad members the knowledge they need to execute.

**Grounded in:** FR-009

## Question (from interview)

AMB-001 (BLOCKING, explicit `[NEEDS CLARIFICATION]`): persist the latest input text
per session/workspace, or per user/global?

## Resolution

Persist the latest input text **scoped to the Copilot session**, keyed by `sessionId`,
in the extension's **own session-scoped artifacts directory under `$COPILOT_HOME`**,
deliberately kept **outside the repository working tree**. When the canvas is reopened
within the same session the text is restored; opening it in a different session shows
no text from another session. Governed by ADR-002. FR-009 is strengthened from *SHOULD*
to *MUST*.

Implementation note for the squad: the SDK provides the `sessionId` and the
extension's artifacts location; `extension.mjs` writes/reads a small per-session record
(e.g. `<artifacts>/<sessionId>/last-input.json`) and the `open` handler seeds the iframe
with the restored text. The persistence write is debounced off the same input-change
path that drives recompute (see MSF-009). No user text is ever written to the repo tree.

## Acceptance criteria

- [ ] After typing text and closing+reopening the canvas **within the same session**,
      the input is restored byte-for-byte (the restored string `===` the last value).
- [ ] The persisted artifact lives under `$COPILOT_HOME` (the extension artifacts dir),
      **not** under the repository working tree — `git status` stays clean after use.
- [ ] Reopening the canvas under a **different `sessionId`** restores **no** text from
      the other session (isolation): the input is empty / that session's own last value.
- [ ] Persistence is keyed by `sessionId`; two sessions editing concurrently never
      observe each other's text.

# ADR-001 — Default the comparison to GA models, preview behind a toggle

- **Status:** accepted
- **Date:** 2026-06-12
- **Affects requirements:** FR-009
- **From facets:** MSF-001

## Context

FR-009 carried a `[NEEDS CLARIFICATION]` marker (interview item AMB-001): when the
Model Cost Comparison Panel first renders, which models should it show — all known
models, only GA models, or only the user's recent models? The decision changes
observable behavior, so it is recorded here rather than only in a facet.

`src/lib/copilotModels.ts` already distinguishes `releaseStatus: "GA" | "Public preview"`
and exposes `COPILOT_MODEL_OPTIONS` (including the synthetic `auto` model), so either
default is cheap to implement.

## Decision

Default the comparison to **GA models only** (plus `auto`). Public-preview models are
hidden behind an explicit, off-by-default "Include preview models" toggle whose state
is persisted alongside the rest of the selection (FR-007).

## Consequences

- The default view is stable and reflects billable, generally-available pricing.
- Users who want preview models opt in explicitly; their choice persists.
- FR-009's acceptance criteria are now measurable (see MSF-001), unblocking its BDD
  scenario and TDD loop entry.
- A future ADR may revisit this if "recent models" telemetry becomes available; this
  ADR would then be superseded rather than edited.
